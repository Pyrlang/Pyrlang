rex module - Remote Execution¶

class pyrlang.rex.Rex¶

Bases: pyrlang.process.Process

Remote executor for RPC calls. Registers itself under the name rex and accepts RPC call messages. Erlang rpc:call sends a $gen_call styled message to the registered name rex on the remote node which we parse and attempt to execute.

See also

gen module documentation and parse_gen_call() function

handle_one_inbox_message(msg) → None¶

Function overrides handle_one_inbox_message() and expects a $gen_call styled message. The result or exception are delivered back to the sender via message passing.

Parameters:msg -- A tuple with Atom $gen_call as the first element
Return type:None
traceback_depth_ = None¶

This being non-zero enables formatting exception tracebacks with the given depth. Traceback is attached as a 'traceback' field in the exception, that is sent to the caller. Default: 5
